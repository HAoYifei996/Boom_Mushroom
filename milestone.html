Current Accomplishments and Results

In the first week, we focused on learning OpenGL and finishing the basic cloth simulation. When we did the basic cloth simulation following the instructions in Project 4, we also learned some basic knowledge about shader, which is pretty important and useful for the further project.

In the second week, we divided the project into two parts. Xinhao was responsible for adding several new features into this simulation system, Shenao and Jieming were responsible for trying to use shaders to calculate the simulation steps, which were calculated in CPU before.

Until now, we have already created shaders for simulation steps based on transform feedback buffer. With the help of GPU's great parallel processing power, we are able to simulate the motion of numbers of vertices at the same time, and achieve a faster and more accurate simulation system. Besides, we have also added some new features, including a moving sphere, a static cube and wind force, and handled a part of collision detection.

Problems to be Solved

Tradeoff between gap and model penetration

As so far, the cloth can interact with the moving ball well in some extent. In order to prevent the ball from going through the cloth model, we decrease the radius when rendering, however, there will be a little gap between the surface of ball and cloth, which looks very strange. If we keep the radius as project4, the ball might go through the cloth. How can we deal with this?

Bouncing on the cube

When I implement the collision between the cube and cloth, it ends with the cloth still slightly bouncing on the cube, which is worse than the situation in self-collisions in project4. Is here any better algorithm for cloth colliding with a plane?
